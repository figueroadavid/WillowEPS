# Import-WEPSXeroxConfig

## Synopsis
Imports Xerox-specific printer configuration from a JSON file and applies it to one or more printers.

## Description
`Import-WEPSXeroxConfig` imports printer configuration data from a JSON file generated by `Export-WEPSXeroxConfig` and writes the exported registry values to one or more target printers.

The function applies values under the printer registry path:

`HKLM\SYSTEM\CurrentControlSet\Control\Print\Printers\<PrinterName>`

For each target printer, the function validates that the printer exists by using `Get-Printer`. If `-RequireXeroxGPD` is specified, the function also validates that the printer uses the `Xerox Global Print Driver PCL6` driver before applying changes.

During import, the function stops the Print Spooler service before writing registry values. If `-RestartSpooler` is specified, the Print Spooler service is started again after processing completes.

The function includes special handling for the `Default DevMode` binary registry value. When that value is imported, the embedded device name is updated to match the target printer name.

Binary registry values are expected to be stored as Base64 strings in the JSON file. Entries with the value `<BINARY_DATA_REDACTED>` are skipped.

The function supports `ShouldProcess`, so `-WhatIf` and `-Confirm` can be used.

## Parameters

### -PrinterName
Specifies one or more printer names to which the configuration will be applied.

Each printer is validated with `Get-Printer`. Printers that are not found are skipped with a warning.

- Type: String[]  
- Required: Yes  
- Accepts pipeline input: No  

### -FilePath
Specifies the path to a JSON file previously created by `Export-WEPSXeroxConfig`.

The file must exist. If the file is not found, the function throws a terminating error.

- Type: String  
- Required: Yes  
- Accepts pipeline input: No  

### -RestartSpooler
Starts the Print Spooler service after the import completes.

If this switch is not specified, the function stops the Print Spooler service but does not explicitly restart it.

- Type: Switch  
- Required: No  

### -RequireXeroxGPD
Limits import operations to printers using the `Xerox Global Print Driver PCL6` driver.

Printers using a different driver are skipped with a warning.

- Type: Switch  
- Required: No  

## Inputs
None

This function does not accept pipeline input.

## Outputs
System.Management.Automation.PSCustomObject

Returns one object per processed printer with the following properties:

- `Printer` - The printer name
- `Success` - Boolean value indicating whether the import succeeded
- `Errors` - Error message text if the import failed

## Examples

### Example
```powershell
Import-WEPSXeroxConfig -PrinterName 'PRN-01' -FilePath 'C:\Temp\PrinterConfig.json' -RestartSpooler -Verbose
```

### Example
```powershell
Import-WEPSXeroxConfig -PrinterName 'PRN-01','PRN-02' -FilePath 'C:\Temp\Config.json' -RestartSpooler
```

### Example
```powershell
Import-WEPSXeroxConfig -PrinterName 'PRN-01' -FilePath 'C:\Temp\Config.json' -RequireXeroxGPD
```

## Notes
* This function performs direct registry modification.
* The Print Spooler service is stopped before registry values are written.
* The Print Spooler service is only started again if -RestartSpooler is specified.
* The function does not provide rollback capability.
* The JSON input file is expected to contain objects generated by `Export-WEPSXeroxConfig`.
* Binary registry values must be Base64-encoded in the JSON file.
* Binary values set to <BINARY_DATA_REDACTED> are skipped.
* The Default DevMode value is modified during import so the embedded printer name matches the target printer.
* Missing target registry paths are skipped.
* Items missing Name or Kind are skipped.
* Printers that cannot be found are skipped with a warning.
* When -RequireXeroxGPD is specified, printers not using `Xerox Global Print Driver PCL6` are skipped.
* The function supports -WhatIf and -Confirm through SupportsShouldProcess.
